0: Introduction¶

The COVID-19 Pandemic is an on-going problem and a threat to our civilization that is yet to be solved. One of the most effective solutions in countries where contact-tracing and isolation result to no avail may be the robust administration of vaccines. Any delays on the administration of COVID vaccines may consequently lead to one too many innocent lives lost. This brings us to the topic of this tutorial, that is, to explore the relationship between COVID case fatality and the administration of vaccination. Divided up into five parts, this tutorial starts with collecting the data from publicly available sources, processing the data sets collected, exploring and analyze the processed data, testing a hypothesis, creating a machine learning model, and finally discussing the insights of our exploration regarding vaccination and COVID death.

2a. Process COVID Cases Data¶

Like what is mentioned above, the JHU's data set has the dates presented as columns. This is a problem because we would like it to be rows of dates in a time-series manner as that is easier to work with. In order to accomplish this, a helper function called extract_metric is written to extract the metrics (i.e. confirm or death) from the raw data frames by basically finding the sum of data from every county in each state, then transpose the portion of data frame so that the dates are presented as rows.

# helper:
#   extract metric from raw data frame as format
#
#     | date | state | value |
#
# params:
#   key_offset=[confirm:0|death:1]
#
def extract_metric(raw, key_offset=0):

  # get list of dates available from source
  dates = [*raw.keys()[(11 + key_offset):].tolist()]

  # compute sum of death by date for each state,
  df = raw.groupby(['Province_State'])[[*dates]].sum()

  # transpose it to get sum of death by state for each date
  df = df.transpose()

  # reset index so date (i.e. index) is a column
  df = df.reset_index()

  # melt death by states into data frame for each date
  df = df.melt(id_vars='index')

  # convert value to int
  df['value'] = df['value'].map(int)

  # consolidate column names
  df = df.rename(
    columns={
      'index'          : 'date',
      'Province_State' : 'state',
      'value'          : 'death' if key_offset else 'confirm'
    }
  )

  # transform dates from string to timestamps
  df['date'] = pd.to_datetime(df['date'], format='%m/%d/%y')

  return df

We will perform an inner merge on the metrics extracted from the confirm and death raw data frames for rows with the same date and state. The implication of an inner merge is that only rows with common date and state are merged, which is what we want because if either does not match (or is missing), then the entry wouldn't be useful anyway.

4: Hypothesis Testing & Machine Learning¶

In these days there are still a portion of vaccines disbelievers spreading misinformations about how the companies and governments are delivering ineffective vaccines to make money. However, in order to let vaccines do its job, vaccination coverage is crucial in stopping the spread of covid and more importantly stopping covid from evolving into more deadly virus. We need to bring up the numbers of vaccinated people to build up herd immunity in communities, so we want to use some sophisticated analysis to show the impact of vaccines to covid cases and deaths for the anti-vaxxers.

We first use linear regression, a machine learning technique, to fit a line for the number of people fully vaccinated versus the number of daily deaths for each states. Here we are using the interaction term to build the linear model. We use interaction terms when we think there are two or more factors that is effecting the output parameter. Here we think that on top of the number of people get fully vaccinated, which states you are living in also change the number of daily deaths due to the political demography of the states, as there seems to be a group of people in certain political party trying to promote that covid and vaccines are a hoax made by their opponent party.

Sklearn has this nice linear regression library that allows users to put in multiple independent variables and do interaction terms. The terms generated above can be interpreted as follows. $$y = c_0*#_fully_vaccinated_100k + c_1*in_Alaska + ... + c_50*fully_vaccinated_100k*in_Alaska + ... + intercept$$ where c are the coefficients generated from above. This is a linear model in high dimension. However, from the linear function above, we can get a linear relationship for each states by getting the right coefficients. Since we now put our states into indicator variable, we can simply pick the coefficient that the variable is related to the state we are interested in. Take Maryland as an example. All variables except _inmaryland and _full_vax_100kinmaryland evaluate to 0. By combining the coeficients for _fully_vaccinated100k, intercept, _inmaryland, and _full_vax_100kinmaryland, we can get the model for Maryland in the form $y = \alpha + \beta * x$. We can use this way to find a line for every states.

Now we have the slope and intercept for the linear model of each state. Recall that our purpose is to find the relationship between the independent variable (fully vaccinated per 100k) and dependent variable (daily death), so we are only going to focus on the slope.

Let us set up hypothesis tests for the relation between the coverage of fully vaccinated people versus daily deaths in each states. We are going to do a model utility test. Our null hypothesis will be that there is no relationship between the two factor and the alternative hypothesis will be there are negative relationship. Also we set our significance level to 0.05. So $$H_0: \beta = 0$$ $$H_a: \beta < 0$$ $$significance\ level = 0.05$$ where $\beta$ represent the slope parameter of the linear model $y = \alpha + \beta x$. The slope represents the change in dependent variable per unit change of the independent variable. In this context, the slope is the change in the number of daily deaths per 1/100k of the population get fully vaccinated.
